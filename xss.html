Segurança em Front-end - XSS

XSS é uma vulnerabilidade que permite que códigos JavaScript sejam executados em uma aplicação através do navegador da vítima. Ela permite o sequestro de sessão do usuário, inserção de conteúdo hostil, roubo de informações pessoais, desfigurar sites ou redirecionar os usuários para páginas maliciosas.

Tipos de XSS

Stored

Caso nossa página tivesse um banco, um script malicioso poderia ser enviado pelo formulário e executado quando outro usuário entrasse na página.

Reflected

Caso nossa página tivesse um servidor, um script poderia ser executado imediatamente na página.

DOM-based

Fora isso, nossa página também poderia ser atacada via o próprio JavaScript, através da manipulação de DOM, sem passar por servidor ou banco de dados.

Meios de mitigação

  • Utilizar a Content Security Policy, que restringe as fontes dos scripts.
  • Determinar padrões de input (apenas números, apenas letras, sem caracteres especiais) e utilizar bibliotecas que bloqueiam scripts maliciosos, feito a DOMpurify.
  • Evitar o uso de innerHTML e optar pelo innerText.
  • Utilizar frameworks e navegadores que já venham com essas mitigações configuradas (ex.: Brave e React).
